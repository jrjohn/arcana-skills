# UX Writing 指南

## 核心原則

### 好文案的特質
| 原則 | 說明 | 範例 |
|------|------|------|
| **清晰** | 一看就懂 | "儲存" vs "執行資料持久化" |
| **簡潔** | 去掉贅字 | "確定刪除?" vs "您確定要刪除此項目嗎?" |
| **有用** | 幫助完成任務 | 提供下一步建議 |
| **一致** | 統一用詞 | 全 App "登入" 不混用 "登錄" |
| **友善** | 像人說話 | "出錯了" vs "Error 500" |

## Voice & Tone

### Voice (品牌聲音)
```
我們的聲音是:
✓ 專業但親切
✓ 自信但不傲慢
✓ 簡潔但不冷淡
✓ 幫助但不說教

不是:
✗ 機械、冰冷
✗ 過度熱情
✗ 賣弄術語
✗ 居高臨下
```

### Tone (語氣調整)
| 情境 | 語氣 | 範例 |
|------|------|------|
| 成功 | 正面、鼓勵 | "太棒了！訂單已送出" |
| 錯誤 | 冷靜、建設性 | "密碼錯誤，請再試一次" |
| 警告 | 認真、直接 | "此操作無法復原" |
| 等待 | 輕鬆、安撫 | "正在處理，請稍候..." |
| 空狀態 | 友善、引導 | "還沒有內容，開始建立吧" |

## UI 文案元素

### 按鈕文字
```
原則: 動詞開頭，說明動作結果

✅ 好:
├── 儲存變更
├── 送出訂單
├── 開始免費試用
└── 新增項目

❌ 避免:
├── 是 / 否 (太模糊)
├── 點擊這裡 (沒意義)
├── 提交 (太技術)
└── OK (不知道會發生什麼)
```

### 標題與說明
```
標題: 名詞或短語，說明是什麼
說明: 補充細節，幫助理解

範例:
┌──────────────────────────┐
│ 雙重驗證              ← 標題
│ 增加一層安全防護，     ← 說明
│ 保護您的帳戶
└──────────────────────────┘
```

### 表單標籤
| 元素 | 規則 | 範例 |
|------|------|------|
| Label | 名詞，不加冒號 | Email |
| Placeholder | 格式提示 | name@example.com |
| Helper | 補充說明 | 我們不會公開您的 Email |
| Error | 問題+解法 | 格式錯誤，請輸入有效 Email |

### 錯誤訊息
```
好的錯誤訊息 = 發生什麼 + 如何解決

❌ 差:
├── Error
├── 發生錯誤
├── 無效輸入
└── 請求失敗

✅ 好:
├── 密碼至少需要 8 個字元
├── 此 Email 已被使用，要登入嗎？
├── 網路連線中斷，請檢查網路
└── 檔案過大，請選擇 10MB 以下
```

## 常用文案範本

### 空狀態
```
[圖示]
尚無 {內容類型}

{鼓勵性說明}

[行動按鈕]

範例:
📭
尚無訊息

當有人傳訊息給您時，會顯示在這裡

[開始對話]
```

### 載入狀態
| 情境 | 短文案 | 長文案 |
|------|--------|--------|
| 一般 | 載入中... | 正在載入內容... |
| 送出 | 送出中... | 正在送出您的請求... |
| 儲存 | 儲存中... | 正在儲存變更... |
| 搜尋 | 搜尋中... | 正在搜尋相關結果... |

### 確認對話框
```
標題: 動作 + 對象
說明: 後果描述
按鈕: 取消 / 確認動作

┌──────────────────────────┐
│ 刪除這篇文章？            │
│                          │
│ 刪除後將無法復原          │
│                          │
│      [取消] [刪除文章]    │
└──────────────────────────┘
```

### 成功訊息
| 動作 | 訊息 |
|------|------|
| 儲存 | 已儲存 |
| 複製 | 已複製到剪貼簿 |
| 送出 | 已送出，我們會盡快回覆 |
| 刪除 | 已刪除 |
| 訂閱 | 訂閱成功！ |

## 術語一致性

### 建立詞彙表
```markdown
## 產品詞彙表

| 術語 | 定義 | 不要用 |
|------|------|--------|
| 登入 | 進入帳戶 | 登錄、Sign in |
| 登出 | 離開帳戶 | 登出、Sign out |
| 儲存 | 保存變更 | 保存、Save |
| 取消 | 放棄操作 | 返回、算了 |
| 刪除 | 永久移除 | 移除、清除 |
| 設定 | 調整選項 | 設置、偏好 |
```

### 避免的用詞
| 避免 | 改用 | 原因 |
|------|------|------|
| 請 | (直接動詞) | 贅字 |
| 您的 | (省略) | 通常可省 |
| 成功/失敗 | 具體描述 | 太籠統 |
| 無效 | 說明原因 | 沒幫助 |
| 錯誤 | 具體問題 | 令人焦慮 |
| 注意 | (直接說) | 通常不必要 |

## 字數限制

### 各元素建議長度
| 元素 | 中文 | 英文 |
|------|------|------|
| 按鈕 | 2-6 字 | 1-3 words |
| 標題 | 5-15 字 | 3-8 words |
| 說明 | 15-40 字 | 10-25 words |
| Toast | 10-20 字 | 5-12 words |
| 錯誤 | 15-30 字 | 8-20 words |

### 多語系空間預留
```
中 → 英: 通常變長 50-80%
中 → 德: 可能變長 100%+
英 → 中: 通常變短 30-50%

預留策略:
├── 按鈕: 寬度留彈性
├── 標籤: 可換行或截斷
└── 表格: 欄寬可調整
```

## 文案審查清單

### 撰寫時
```
□ 用使用者的語言，非技術術語
□ 動詞開頭的行動文案
□ 說明動作的結果
□ 錯誤訊息提供解法
□ 一致使用詞彙表術語
□ 去掉「請」「您的」等贅字
□ 符合字數限制
```

### 審查時
```
□ 大聲朗讀，聽起來自然嗎？
□ 使用者能立即理解嗎？
□ 是否有更簡短的說法？
□ 語氣符合情境嗎？
□ 與其他畫面一致嗎？
□ 翻譯後空間夠嗎？
```
