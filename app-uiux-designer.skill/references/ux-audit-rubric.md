# UX Audit Rubric — 設計依據 + 稽核檢核表

用途:①**設計時**(uiux 節點產 `uiuxSpec`)依此做出好 UX;②**稽核時**(PM 節點可用性維度)依同一份檢核表抓爛 UX。兩邊共用一套標準,避免「設計一套、驗另一套」。補足本 skill 原本偏薄的**閱讀心理學 / 認知負荷 / 互動可供性**。

---

## 1. 視覺層次 (Visual Hierarchy) — 反「等權重堆疊」

**原則**:任何畫面必須有明確的 primary / secondary / tertiary 層級,讓眼睛知道「先看哪、再看哪」。

- ❌ **反樣式:等權重 N 等分堆疊** — 把內容平均塞進 4 等分(或多格)、每格同樣大同樣重,使用者不知從何看起、怎麼用。這是最常見的「技術上有畫面、實際沒法用」。
- ✅ 一個明確主區(最大/最上/最亮),其餘為輔;用大小、對比、留白、位置建立層級。
- 依據:Gestalt(見 `gestalt-psychology.md` 接近性/相似性/共同區域)、視覺權重。

## 2. 互動可供性 (Interaction Affordance) — 收合 / 漸進揭露

**原則**:密集的工具/資訊不可一次全塞給使用者;給控制權與焦點。

- ✅ **工具列 / 側面板可收合展開 (collapse/expand)** — 密集工具區必須能縮起,讓使用者專注內容。
- ✅ **漸進揭露 (progressive disclosure)** — 進階/次要選項預設收起,需要時才展開(「更多」、可展開段落、分頁精靈)。
- ✅ **響應式密度** — 視窗/裝置變窄時面板自動收合或改抽屜。
- ❌ 反樣式:所有工具、所有欄位、所有面板永遠同時展開,無法縮、無主次。

### 2b. 版面欄位預算 (Column Budget) — 反「多欄擠壓」

**原則**:水平寬度是稀缺資源。一個 feature 內容區桌機**同時最多 2–3 欄**;**自帶多欄的嵌入式編輯器(form-js / bpmn-js:palette|畫布|屬性)算它自己的 3 欄** → host 這種編輯器的頁面**不得在旁邊再放常駐設定欄**,編輯器要吃滿寬。設定→頂部 toolbar,次要→漸進揭露 drawer。

- ❌ **反樣式**:常駐設定欄擠壓自帶多欄的編輯器(`nav + 設定欄 + 編輯器3欄 = 5 格`,畫布餓死)。
- 完整版面機制(4 種原型 / column budget 算法 / 前後案例 / 可量測門檻)見 **`layout-workspace-patterns.md`**。

## 3. 認知負荷 (Cognitive Load)

- **Miller 7±2**:單一決策點的選項/欄位不宜超過 ~7;超過就分組、分步、分頁。
- **Hick's Law**:選項越多決策越慢——精簡選項、給預設、突出建議動作。
- **一次一個主要任務**:每個畫面聚焦一個主目標,別讓使用者同時面對多個對等任務。

## 4. 掃描動線 (Scan Path / 閱讀心理學)

- **F-pattern**(文字密集/清單):重點資訊靠左上與各段開頭;主要動作在掃描路徑上。
- **Z-pattern**(landing/簡單頁):視線走左上→右上→左下→右下,主 CTA 放路徑端點。
- ❌ 反樣式:主要動作藏在角落/摺線下/掃描死角。

## 5. 目標尺寸 (Fitts's Law)

- 主要動作要夠大、夠近、不擁擠;觸控目標 ≥ ~44×44px;高頻動作放好按到的位置(邊角/固定列)。

## 6. 可及性 (Accessibility / WCAG)

- 文字對比 ≥ 4.5:1(大字 3:1);完整**鍵盤操作路徑**與可見 focus;控件有 label / aria;不倚賴顏色單一傳達;詳見 `accessibility.md`。

## 7. 狀態完整性 (States)

- 每個非同步 / 清單 / 表單畫面都要有:**空狀態**(引導下一步)、**載入中**、**錯誤**(可讀訊息 + 復原路徑)、成功回饋。

---

## 稽核判定(PM 用)

user-facing UI **未過可用性** 且需 **NOGO + 具體回饋** — 若違反上述任一且可客觀指出:
- 等權重 N 等分堆疊、無視覺層次
- **超出 column budget:常駐設定欄擠壓自帶多欄的嵌入式編輯器(nav+設定+編輯器3欄=5 格)**(見 `layout-workspace-patterns.md`)
- 密集工具列/面板不可收合、無漸進揭露
- 單畫面認知負荷過重(選項/欄位爆量、多對等任務)
- 主動作不在掃描動線上
- 目標過小/擁擠(Fitts)
- WCAG 對比/鍵盤/label 不足
- 缺空/載入/錯誤狀態

**過** = 明確層級 + 密集處可收合/漸進揭露 + 在認知限度內 + 主動作在掃描動線 + 目標尺寸足 + a11y + 狀態齊。

**只有真正主觀/品牌/美感取捨**(配色偏好、插畫風格、文案語氣的品牌調性)→ 由 PM 判 **HOLD 交人**,不當 NOGO 硬性擋。
