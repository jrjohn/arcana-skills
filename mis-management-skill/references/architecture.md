## Why a central hub

Putting cron jobs / scripts on the hub (instead of distributed across each device) gives:

1. **Single host to harden** — only one box needs firewall rules, log retention, intrusion detection.
2. **Modern toolchain available** — even if NAS has 2018 OpenSSL, hub has TLS 1.3.
3. **Single SSH-key chain** — hub holds the keys to talk to AD / NAS / switches; one ssh-agent unlocks all.
4. **Logs aggregated** — `rsyslog` fan-in: FG / switches / NAS all syslog to hub, split by source IP into per-device files.
5. **Easy to back up** — one `tar` of `/opt/mis-*` + `/etc/cron.d/mis-*` + `/etc/rsyslog.d/` captures the whole MIS state.

## Hub directory layout

```
/opt/mis-log-db/                   # all scripts + SQLite DB
├── *.py / *.sh                    # cron-driven scripts
├── mis-log.db                     # SQLite DB (FG forward log aggregation)
├── ip-names.tsv                   # generated by gen-ip-names-tsv.py
├── ip-names-manual.tsv            # manual override (infra labels)
└── *.bak-YYYYMMDD-HHMMSS          # ad-hoc backups before risky changes

/opt/mis-http/                     # Apache web root (port 8081)
├── index.html
├── threatfeed.txt                 # generated by mis-threatfeed-sync (FG fetches via HTTP)
├── threatfeed-manual.txt          # human-curated additions
├── threatfeed-manual-auto.txt     # auto-curated additions
├── ai-usage.html                  # generated by gen-ai-usage-report.py
├── deny-report.html               # generated by gen-deny-report.sh
└── dns-ai-counts.json             # cached DNS-AI events from AD DC

/var/log/mis-*.log                 # cron job logs
/var/log/remote/                   # rsyslog fan-in
├── fortigate/fortigate.log        # FG syslog
├── switch/<sysname>.log           # per-switch syslog
└── qnap/qnap.log                  # NAS syslog

/etc/cron.d/mis-*                  # cron entries
├── mis-log-aggregate              # */10 min — FG log → SQLite
├── mis-gen-ip-names               # hourly :02 — NetBox+FG → ip-names.tsv
├── mis-auto-curate-threatfeed     # 15,45 — scan FG log → manual-auto.txt
├── mis-threatfeed-sync            # 20,50 — merge sources → threatfeed.txt
└── mis-nas-clamav-update          # daily 04:30 — ClamAV sigs proxy → NAS

# NOT in /etc/cron.d/ (was historically in root crontab):
# 25,55 — threatfeed-to-fg-addrgrp.py — push threatfeed.txt → FG addrgrp
```

## Data flow examples

### Inbound attacker block flow

```
1. Attacker IP probes WAN
        ↓
2. FG default deny (or local-in policy 5/6) → log to syslog
        ↓
3. FG syslog → rsyslog on hub → /var/log/remote/fortigate/fortigate.log
        ↓
4. :15/:45 cron: auto-curate-threatfeed.py greps last 1h log
        ↓
5. If srcip > 100 hits/h → append to threatfeed-manual-auto.txt
        ↓
6. :20/:50 cron: mis-threatfeed-sync-v3.py merges all sources → threatfeed.txt
        ↓
7. :25/:55 cron: threatfeed-to-fg-addrgrp.py PUTs to FG REST API
        ↓
8. FG addrgrp Threatfeed-Inbound-Auto-{0..7} updated
        ↓
9. FG local-in-policy 5/6 referencing parent addrgrp → blocks the IP

Worst-case latency: ~30 min from first hit → blocked.
```

### IP labeling flow

```
NetBox IP record (.113 + assigned to contact "Janey")
                ↓ API call
             gen-ip-names-tsv.py  (cron hourly :02)
                ↓                                 ↓
       FG DHCP description                 ip-names-manual.tsv (infra)
                ↓                                 ↓
                └──────── merge by IP ────────────┘
                                ↓
                     /opt/mis-log-db/ip-names.tsv
                                ↓
        gen-ai-usage-report.py / gen-deny-report.sh
                                ↓
                     HTML reports show "Janey" not ".113"
```

### Hub host requirements

- **OS**: CentOS 8 / Stream 8 / RHEL 8 / Rocky 8 (or any modern Linux with the listed packages)
- **CPU**: 2 cores enough; 4+ better if running LibreNMS too on same host
- **RAM**: 4 GB minimum
- **Disk**: 100 GB for log retention (FG syslog ~30 GB/year at typical SMB volume)
- **Net**: 1 Gbps to LAN; needs IP routing to FG management interface + outbound to clamav.net / threatfeed providers
