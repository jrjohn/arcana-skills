#### STP-001 單元測試策略

| 屬性 | 內容 |
|-----|------|
| **ID** | STP-001 |
| **名稱** | 單元測試策略 |
| **目的** | 驗證個別軟體單元的正確性 |
| **對應標準** | IEC 62304 §5.5.5 |
| **適用分類** | Class B, Class C |

**測試方法**：
- 白箱測試：語句覆蓋、分支覆蓋、條件覆蓋
- 黑箱測試：等價類劃分、邊界值分析

**覆蓋率要求**：
| 安全分類 | 語句覆蓋 | 分支覆蓋 | MC/DC |
|---------|---------|---------|-------|
| Class A | - | - | - |
| Class B | ≥ 80% | ≥ 70% | - |
| Class C | ≥ 90% | ≥ 85% | ≥ 80% |

**測試工具**：
| 工具名稱 | 用途 | 版本 |
|---------|------|------|
| [工具1] | 單元測試框架 | [版本] |
| [工具2] | 覆蓋率分析 | [版本] |

#### STP-002 整合測試策略

| 屬性 | 內容 |
|-----|------|
| **ID** | STP-002 |
| **名稱** | 整合測試策略 |
| **目的** | 驗證軟體單元整合後的正確性 |
| **對應標準** | IEC 62304 §5.6 |
| **適用分類** | Class B, Class C |

**整合方式**：
- [ ] 大爆炸整合 (Big Bang)
- [x] 漸進式整合 (Incremental)
  - [x] 由上而下 (Top-down)
  - [ ] 由下而上 (Bottom-up)
  - [ ] 三明治 (Sandwich)

**整合順序**：
```
Layer 1: Presentation → Business (SDD-001 → SDD-002)
Layer 2: Business → Data (SDD-002 → SDD-003)
Layer 3: 外部介面整合 (SDD-003 → External)
```

**測試項目**：
| 項目 | 說明 | 對應模組 |
|-----|------|---------|
| 介面正確性 | 模組間介面呼叫正確 | SDD-001, SDD-002 |
| 資料流正確性 | 資料傳遞正確無誤 | SDD-002, SDD-003 |
| 錯誤傳遞 | 錯誤正確傳遞處理 | 全模組 |

## 6. 風險評估

### 6.1 測試風險

| 風險 ID | 風險描述 | 可能性 | 影響 | 緩解措施 |
|--------|---------|--------|------|---------|
| RISK-001 | 測試環境不穩定 | 中 | 高 | 備援環境、環境監控 |
| RISK-002 | 測試資源不足 | 中 | 中 | 優先級排序、外部支援 |
| RISK-003 | 需求變更頻繁 | 高 | 高 | 變更管理流程 |
| RISK-004 | 缺陷修復延遲 | 中 | 高 | 缺陷追蹤、每日站會 |

### 6.2 缺陷管理

#### 6.2.1 缺陷分類

| 嚴重程度 | 定義 | 回應時間 |
|---------|------|---------|
| Critical | 系統崩潰、資料遺失、安全漏洞 | 立即 |
| High | 主要功能無法使用 | 24 小時 |
| Medium | 功能異常但有替代方案 | 3 天 |
| Low | 外觀或次要功能問題 | 下一版本 |

#### 6.2.2 缺陷流程

```
發現 → 回報 → 確認 → 分派 → 修復 → 驗證 → 結案
```
